CSS Selectors & Specificity

Description Example Precedence Considerations

The universal selector (*) applies a CSS rule to all html elements.

 * { margin: 0; }

/* sets margin of all html elements to zero */

None

This selector should be avoided, and instead more specific selectors should be used.

The type selector applies a CSS rule to specified html elements.

 body { margin: 10 px; }

/* sets margin of body element to 10 px */



h1, h2, p {border: 1px solid black;}

/* defines a border around all <h1>, <h2>, and <p> elements */

Overrides the universal selector

The second example shows how this selector can be used to avoid repetitive code.

The class selector applies a CSS rule to all html elements with a class attribute having the same value (the same class name). A period is placed before the class name.

 <p class="green">This is paragraph 1.</p>

<p>This is <span class="green">paragraph 2.</span></p>


.green {color: green; }

/* sets the color of the text in paragraph 1 and text within span tags in paragraph 2 to green, but does not affect other text in paragraph 2.*/

Overrides the type and universal selectors

This selector makes code writing more efficient because the same style rule can be applied to more than one element. An html element can have more than one class.

The descendant combinator selector applies a CSS rule to nested html elements or to an element within a class.

 <p class="red">This paragraph has the same class name as the <a> tag below, but will not be colored red.</p>

<ul>
<li class="decorate"><a class="red">click me</a>
</li>
<li><a>don"t click me</a></li>
<ul/>

li { text-decoration: underline; }

/* underlines text within the <li> tag*/


li a.red { color: red; }

/* only <a> tag with class name of "red" gets red text, but text in paragraph with class name "red" is not affected */

Overrides type rule of parent element or type rule for child elements

The last example given demonstrates the use of "chaining," where multiple selectors are combined, allowing for greater specificity.

The id selector (#) applies a CSS rule to an element with the named id value.

 <div id="container">[content of division]</div>

<div>[content of division]</div>

#container { background-color: gold;}

/* sets the background-color to gold of the div element with the id name of "container" */

Overrides combinator, class, type, and universal selectors

Id values are unique. No two elements on a page can have the same id value.

The attributes selector applies a CSS rule to an html element with the specified attribute.

 <a href="index.html">I am an html file.</a>

[href]{ text-transform: uppercase; }

/*text within <a> tag is made uppercase*/

Overrides class, type, and universal selectors

Other attributes like src could be used as CSS selectors. This selector also allows for searching for an element by its attribute value.

The psuedo-classes selector (written as a colon followed by a name) applies CSS rules to affect the different states of an element, such as when a user hovers or clicks on an element.

 <p>Hover over me!</p>

p:hover { color: lime; }

/*changes the color of the paragraph to lime on hover. Try it!*/

The specificty of this selector is self-defined.

This selector applies to any element. Other states of elements include focus, visited, disabled, active.

The important selector (!important) applies a CSS rule of the highest priority.

 <h1 id="priority">This is important!</h1>

#priority { font-size: 24px; text-decoration: underline; }

/* sets the font-size to 24 pixels and underlines the <h1> heading with the id name of "priority" */



<h1>This is important!</h1>

h1 { font-size: small !important; font-style: italic; }

/* overrides the id (#priority) rule and sets the font-size to small and italic*/

Overrides all other CSS rules

This selector should be avoided because it disrupts the rules of specificity, and the rules for overriding it are complex.

 <p class="red">I will be green, not red.</p>
